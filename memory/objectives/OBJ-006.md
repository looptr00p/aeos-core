# Objective

**Traceability ID**: OBJ-006

## Title

Repository Truth Reconciliation — Documentation and Structural Remediation

## Status

DRAFT

## Description

Remediate critical and medium-severity inconsistencies identified during the repository truth reconciliation audit (2026-05-19). This objective addresses documentation accuracy, structural correctness, CI completeness, and traceability index integrity — without modifying governance rules, protocols, or operational memory artifacts.

## Strategic Purpose

AEOS Core's governance-first model depends on repository truth being the single source of truth. When documentation claims diverge from actual repository state, the foundational "repo-driven" principle is undermined. This objective restores alignment between declared state and actual state, ensuring that onboarding, audit, and operational decisions are based on accurate information.

## Problem Statement

The repository truth reconciliation audit identified 18 inconsistencies and 6 structural observations. Three are HIGH severity:
1. README.md repository tree diagram is structurally incorrect (shows all directories under `aeos-core/` when they are at root level).
2. ESTADO_DEL_ARTE.md contains the same incorrect tree plus stale metrics.
3. CI workflow runs only `aeos-core/tests`, missing 11 root-level test files that validate governance, protocols, templates, agents, and workflows.

Five MEDIUM-severity gaps further degrade trust in documentation: review directory mapping error in traceability index, version mismatch across artifacts, stale metrics, semantic misnaming of `memory/research/`, and phase designation misalignment.

## Desired Outcome

All HIGH and MEDIUM severity gaps from the reconciliation audit are resolved. Documentation accurately reflects repository state. CI validates the complete test suite. Traceability index is internally consistent.
