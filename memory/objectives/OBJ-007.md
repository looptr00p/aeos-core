# Objective

**Traceability ID**: OBJ-007

## Title

Governance Enforcement Layer — From Declarative to Operationally Enforceable

## Status

DRAFT

## Description

Transform AEOS governance from declarative (documented rules without enforcement) to operationally enforceable (rules that are validated, tracked, and auditable). This objective introduces deterministic governance checks, approval record artifacts, lifecycle transition validation, and CI-based enforcement — without adding autonomous systems, runtime orchestration, or hidden state.

## Strategic Purpose

AEOS Core has comprehensive governance documentation (7 governance files, 7 protocols, 11 templates, 6 workflows, 7 agents). However, governance is partially aspirational: approvals are not recorded, ADR requirements are not enforced, lifecycle transitions are weakly validated, and CI does not validate governance integrity. This objective closes the gap between declared governance and operational enforcement.

## Problem Statement

The governance review (REV-010) scored AEOS at 1.8/4.0, below the acceptable threshold of 2.0. The primary weaknesses are:

1. **No approval records**: Human approval is required by governance but no artifact records that approval occurred.
2. **No ADR enforcement**: ADR_PROTOCOL.md requires ADRs for governance changes, but no check validates compliance.
3. **CI does not validate governance integrity**: CI runs tests for file existence but does not check governance compliance (approvals, ADRs, lifecycle transitions).
4. **Lifecycle transitions are not validated**: Tasks move between states without recording when, by whom, or whether prerequisites were met.
5. **Version declarations are inconsistent**: Three artifacts declare different versions with no enforcement mechanism.

These gaps mean governance rules exist as documentation but are not operationally enforced.

## Desired Outcome

All governance rules that can be deterministically validated are enforced through automated checks. All human approvals are recorded as explicit artifacts. All lifecycle transitions are tracked. All governance violations are detected and reported.

## Constraints

- Must preserve human authority as final decision maker
- Must remain markdown-first and repository-driven
- Must remain locally executable (no cloud dependencies)
- Must remain deterministic (no probabilistic reasoning)
- Must not introduce hidden state or implicit memory
- Must not add more than 15 new lint checks total (currently 11, adding up to 4 more in this objective)
- Must not modify existing governance document substance
- Must not break existing tests or lint checks
- Must remain maintainable by a single human operator
- All enforcement checks must be auditable and explainable
