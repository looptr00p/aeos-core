# Objective

**Traceability ID**: OBJ-003

## Title

Prepare AEOS for real engineering project usage

## Status

ACTIVE

## Description

Transition AEOS from simulation validation to real engineering project usage. This objective focuses on operational ergonomics refinement to reduce friction discovered during external operational validation (OBJ-002), while preserving all governance invariants.

AEOS has completed internal operational validation (OBJ-001) and external operational simulation validation (OBJ-002). This objective prepares the system for actual engineering work on real external projects.

## Business Context

Operational validation cycles revealed that AEOS governance structure is sound but creates operational friction in three areas: workflow stage count for simple changes, review type proliferation, and manual traceability maintenance. This objective addresses these ergonomics issues through guidance documents, templates, and lightweight mechanical validation — without weakening governance invariants.

## Success Criteria

- Workflow tiering guidance created and operational
- Review minimization guidance created and operational
- Traceability assistance template created
- Operational ergonomics document completed
- Lightweight traceability validation added to aeos_lint.py
- All governance invariants preserved (no weakening)
- No orchestration or autonomous behavior introduced
- aeos_lint.py and pytest pass

## Constraints

- Must not weaken governance invariants.
- Must not introduce orchestration or autonomous behavior.
- Must not add dependencies beyond pytest and pyyaml.
- Must remain markdown-first and repo-driven.
- Human governance boundary must be preserved.
