# PHASE_POLICY.md

## AEOS Core Phases

AEOS Core evolves through defined phases. Each phase has strict entry and exit criteria.

### Phase 0: Governance/Bootstrap

**Current Phase: AEOS Core v0 starts in Phase 0.**

- Purpose: Establish governance, protocols, templates, and baseline structure.
- Allowed: Creating governance documents, protocols, templates, agent definitions, workflows.
- Forbidden: Production workloads, external integrations, autonomous operations.
- Exit Criteria: All governance files created, all protocols defined, all templates ready, tests passing.

### Phase 1: Controlled Internal Adoption

- Purpose: Validate AEOS Core within a single project or team.
- Entry Criteria: Phase 0 complete, human approval to proceed.
- Allowed: Internal task execution, workflow validation, agent operation within scope.
- Forbidden: Multi-project usage, external exposure, platformization.
- Exit Criteria: Successful completion of multiple tasks, audit validation, no governance violations.

### Phase 2: Operational Multi-Project Usage

- Purpose: Validate AEOS Core across multiple projects or teams.
- Entry Criteria: Phase 1 complete, human approval to proceed, documented success metrics.
- Allowed: Cross-project workflows, shared governance, scaled agent operations.
- Forbidden: Platformization, external API exposure, autonomous scaling.
- Exit Criteria: Repeated successful multi-project validation, consistent audit results.

### Phase 3: Possible Platformization

- Purpose: Evaluate platformization ONLY after repeated validation.
- Entry Criteria: Phase 2 complete, human approval to proceed, documented repeated validation.
- Allowed: Platform evaluation, external integration proposals, scaling architecture review.
- Forbidden: Autonomous platformization, uncontrolled expansion, self-modifying governance.
- Note: Phase 3 does NOT guarantee platformization. It only permits evaluation.

### Phase Transitions

- Phase transitions REQUIRE human approval.
- Phase transitions REQUIRE documented justification.
- Phase transitions REQUIRE audit review.
- No agent may initiate a phase transition autonomously.
