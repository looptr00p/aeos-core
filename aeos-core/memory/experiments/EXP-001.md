# EXP-001

## Hypothesis

Defining `aeos-core` as an Experimental Governance Lab improves governance clarity and reduces accidental canonical assumptions.

## Context

AEOS canonical operations are now centered in `aeos-core-opencode`.
This repository is used for controlled governance and validation experiments.

## Scope

- repository role clarification
- controlled porting governance artifacts
- experiment tracking artifacts

## Non-Goals

- canonical operational changes
- runtime orchestration capabilities
- cross-repo automation

## Artifacts Created

- docs/EXPERIMENTAL_LAB_CHARTER.md
- governance/PORTING_POLICY.md
- templates/experiment_template.md
- templates/porting_review_template.md
- memory/experiments/PORT-001.md

## Validation Plan

- run `python3 aeos-core/scripts/aeos_lint.py`
- run `cd aeos-core && pytest`

## Risks

- role confusion if charter/policy are ignored
- premature porting attempts without review discipline

## Expected Benefit

Clear separation between experimental and canonical governance workflows.

## Porting Relevance

Provides governance artifacts that can be considered for canonical adoption only after explicit review and human approval.
